Задание разрешений для общего ресурса

Общими ресурсами называют любые ресурсы, доступные для пользователей сети: каталоги, файлы, принтеры и именованные каналы. Это также относится к находящимся на сервере ресурсам, доступным для пользователей сети. При совместном использовании ресурса вместо разрешений NTFS применяются общие разрешения.

Важно!

Разрешения для общего ресурса применяются только к пользователям, доступ которых к ресурсу осуществляется по сети. Они не применяются к пользователям, входящим в систему локально, как на сервере терминалов. Чтобы запретить доступ к объектам для пользователей, входящих в систему локально, следует установить разрешения NTFS на вкладке Безопасность страницы Свойства объекта.

В зависимости от типа ресурса существуют два способа установить разрешения для общего ресурса.

Задание разрешений для файла или папки с помощью мастера общего доступа к файлам

  1. Щелкните правой кнопкой мыши файл или папку и в контекстном меню выберите команду Общий доступ.

  2. Следуйте инструкциям мастера по выбору пользователя и группы для предоставления общего доступа к файлу или папке, а также для задания разрешений для файла или папки для каждого пользователя или группы.

Задание разрешений для ресурса с помощью проводника Windows

  1. Откройте проводник Windows.

  2. Правой кнопкой мыши щелкните объект и выберите пункт Общий доступ или Свойства.

  3. Откройте вкладку Общий доступ и щелкните Дополнительные параметры доступа, чтобы задать разрешения.

Прочие вопросы

  • Чтобы открыть проводник Windows, нажмите кнопку Пуск, последовательно выберите Все программы, Стандартные, а затем - Проводник.

  • Мастер общего доступа к файлам используется для управления общими ресурсами как на локальных, так и на удаленных компьютерах. С помощью проводника Windows и командной строки можно управлять только общими ресурсами на локальном компьютере.

  • При комбинированном использовании разрешений для общего ресурса и разрешений на уровне файловой системы всегда применяются более строгие разрешения.

  • Обычно проще назначать разрешения группам, а затем добавлять пользователей в эти группы, чем назначать одинаковые разрешения нескольким отдельным пользователям.

  • Для измененных разрешений на особых общих ресурсах, таких как ADMIN$, могут быть восстановлены параметры по умолчанию при остановке и перезапуске службы сервера или при перезапуске компьютера. Это не относится к созданным пользователями общим ресурсам, имена которых оканчиваются знаком «$».
