Просмотр действующих разрешений для файлов и папок

При создании файла или папки Windows для этих объектов устанавливает разрешения по умолчанию, либо их создатель может установить особые разрешения.

Чтение - это минимальное разрешение, необходимое для просмотра действующих разрешений. Подробные сведения об этом можно получить в подразделе «Прочие вопросы».

Чтобы просмотреть действующие разрешения для файлов и папок

  1. Откройте проводник Windows и найдите файл или папку, для которых требуется просмотреть действующие разрешения.

  2. Щелкните файл или папку правой кнопкой мыши, выберите пункт Свойства и щелкните вкладку Безопасность.

  3. Нажмите кнопку Дополнительно, щелкните вкладку Действующие разрешения и нажмите кнопку Выбрать.

  4. В поле Введите имя выбираемого объекта (примеры) введите имя пользователя или группы и нажмите кнопку ОК. Установленные флажки соответствуют действующим разрешениям на доступ пользователя или группы к данному файлу или папке.

Прочие вопросы

  • Чтобы открыть проводник Windows, нажмите кнопку Пуск, последовательно выберите Все программы, Стандартные, а затем - Проводник.

  • Если доступ к указанному объекту предоставлен группе «Все», группе «Проверенные пользователи» или группе «Локальные пользователи», эти разрешения всегда входят в действующие права, кроме случаев, когда указанный пользователь или группа являются членами группы «Анонимные пользователи». В данной версии Windows группа «Анонимные пользователи» не входит в группу «Все».

  • На вкладке Действующие разрешения показаны данные, вычисляемые на основе записей об имеющихся разрешениях. Поэтому содержимое этой страницы доступно только для чтения; разрешения пользователя нельзя изменять, устанавливая или снимая флажки разрешений.

  • Разрешения устанавливаются только на дисках формата NTFS при использовании пользовательского интерфейса управления доступом.

  • Разрешения для общего ресурса также не учитываются при вычислении действующих разрешений. Доступ к общим папкам может быть отклонен по разрешениям для общих ресурсов, даже если доступ разрешен разрешениями NTFS.
